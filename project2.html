Predicting Bankruptcy
Using Machine Learning

Type
Classification · ML
Model
Random Forest Classifier
Libraries
Pandas · NumPy · Scikit-learn · Matplotlib · Seaborn
01 — Overview
Objective

The goal of this project was to develop a machine learning model to predict whether a company would go bankrupt based on financial indicators. The project involved preprocessing an imbalanced dataset, training a model, and evaluating its performance to achieve high accuracy and reliability.

This solution has potential applications in financial risk assessment and early warning systems for investors, creditors, and regulatory stakeholders.

02 — Data
Dataset Overview
Target Variable
Bankrupt? — binary classification: 1 for bankruptcy, 0 for no bankruptcy
Features
Financial attributes: debt ratio, working capital, profitability metrics, and more
Key Challenge
Significant class imbalance — far more non-bankrupt companies than bankrupt ones
Split Ratio
80/20 train/test split using train_test_split from Scikit-learn
03 — EDA
Exploratory Data Analysis
Class balance visualization — revealing significant imbalance in the target variable

Class imbalance was identified early as a critical issue. With significantly more non-bankrupt companies, a naive model would simply predict "not bankrupt" for everything and still score high accuracy — making it useless in practice.

Solution: RandomOverSampler was applied to the training set, ensuring both classes were equally represented before model training.

04 — Model
Model Performance
Confusion matrix — evaluating true positives, false positives, and false negatives
1302

16

False Positives
  • Excellent non-bankruptcy detection — 1302 true negatives and only 11 false positives, indicating a very low false positive rate for healthy companies.
  • Bankruptcy recall needs improvement — 16 true positives vs. 35 false negatives. The model misses some bankrupt companies, which is a known tradeoff with imbalanced data.
  • Practical value remains high — the model reliably clears healthy companies, minimizing unnecessary alarm while still flagging a meaningful portion of risky ones.
05 — Features
Feature Importance
Feature importance — identifying the most critical financial metrics driving bankruptcy prediction

Feature importance analysis reveals which financial indicators have the most predictive power. This insight is valuable beyond the model itself — it tells analysts which financial ratios to watch most closely when assessing company health.

06 — Deployment
Model Saving

The trained Random Forest model was serialized using Python's pickle library, making it ready for future deployment, API integration, or embedding into a financial risk monitoring system.

07 — Conclusion

This project successfully developed a Random Forest classifier capable of predicting bankruptcy with high accuracy, effectively handling class imbalance through oversampling and identifying key financial indicators driving predictions. The solution has clear applications in financial risk assessment and early warning systems for lenders, investors, and regulators.
